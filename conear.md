# 词项聚集
用来描述很多词共同临近出现。

## 语法
```
(A | B | C | ...)//distance*arity/freq/
```

该表达式为真当且仅当以下条件同时为真时：

1. 列表中的词项连续出现构成一个 `near/+distance` 序列，即相邻词项之间的距离小于等于 distance
1. 这样序列中词项数量大于等于 arity
1. 这样的序列的数量大于等于 freq

例如：

```
(律师|北京|医院|医疗|纠纷|医生|病情|事故)//20*4/3/
```

## 与 {filter} 协同

```
(律师|北京|医院|医疗|纠纷|医生|病情|事故){filter}//20*4/3/
```
表示这个列表中的所有词都仅用作验证，不作为召回入口。

```
((律师|北京|医院|医疗|纠纷|医生){filter}|病情|事故)//20*4/3/
```
仅 `{filter}` 修饰的部分不作为召回入口，无 `{filter}` 修饰的 `病情|事故` 是召回入口的候选（有可能败选从而也不是召回入口）。

## 优先级
该算符 `//` 属于后缀算法，优先级最高，高于作为一元算符的 `not`，当然，`//` 算符操作的表达式不能包含 `not`，例如：

```
# 合法表达式
法院 and not (律师|北京|医院|医疗|纠纷|医生|病情|事故)//20*4/3/
# 等效于
法院 and not ((律师|北京|医院|医疗|纠纷|医生|病情|事故)//20*4/3/)
```

```
# 非法表达式，"//" 算符只能作用在 or 表达式上
法院 and (not (律师|北京|医院|医疗|纠纷|医生|病情|事故))//20*4/3/
```
